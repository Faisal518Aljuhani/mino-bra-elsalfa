# نشر الموقع على الإنترنت (Render.com) 🌐

عشان أي شخص يقدر يفتح اللعبة برابط، لازم يشتغل السيرفر على استضافة حقيقية (تفضل شغّالة حتى لو جهازك مقفل). **Render** يوفر خطة مجانية تكفي هذا المشروع.

## الخطوات

### 1) ارفع المشروع على GitHub
1. أنشئ حساب مجاني في https://github.com (لو ما عندك)
2. أنشئ مستودع (Repository) جديد، خله اسمه مثلاً `mino-bra-elsalfa`
3. ارفع كل مجلد المشروع (`backend` و `frontend` و `README.md`) داخل المستودع
   - أسهل طريقة: من صفحة المستودع اضغط "Add file" → "Upload files" واسحب المجلد كامل

⚠️ **مهم:** لا ترفع ملف `.env` أبداً (فيه أسرارك). المشروع فيه ملف `.gitignore` يمنع رفعه تلقائياً لو استخدمت git من الطرفية، بس لو رفعت يدوياً من المتصفح انتبه لا تسحبه.

### 2) أنشئ حساب Render واربطه بـ GitHub
1. روح لـ https://render.com وسجل حساب (تقدر تسجل مباشرة بحساب GitHub)
2. اضغط **New +** → **Web Service**
3. اختر المستودع اللي رفعته

### 3) إعدادات الخدمة
عبّي هذي الحقول بالضبط:
| الحقل | القيمة |
|---|---|
| Root Directory | `backend` |
| Runtime | Node |
| Build Command | `npm install` |
| Start Command | `npm start` |
| Instance Type | Free |

### 4) متغيرات البيئة (Environment Variables)
في نفس صفحة الإعداد، روح لقسم **Environment** وأضف نفس المتغيرات اللي بملف `.env` عندك محلياً:

| Key | Value |
|---|---|
| `JWT_SECRET` | نص عشوائي طويل وسري |
| `SMTP_HOST` | `smtp.gmail.com` |
| `SMTP_PORT` | `465` |
| `SMTP_SECURE` | `true` |
| `SMTP_USER` | إيميلك |
| `SMTP_PASS` | App Password |
| `APP_BASE_URL` | بعد أول نشر، Render بيعطيك رابط مثل `https://mino-bra-elsalfa.onrender.com` — ارجع وحط نفس هذا الرابط هنا |

*(لا تضيف `PORT` — Render يحدده تلقائياً بنفسه)*

### 5) اضغط Create Web Service
انتظر شوي (أول مرة تاخذ دقيقتين تقريباً) لحد ما تشوف "Live" باللون الأخضر.

### 6) الرابط جاهز!
بيطلع لك رابط شكله:
```
https://mino-bra-elsalfa.onrender.com
```
هذا الرابط تقدر ترسله لأي شخص، ويفتح اللعبة من جواله أو جهازه مباشرة.

## ملاحظات مهمة

- **الخطة المجانية تنام بعد عدم الاستخدام**: لو ما فيه أحد يفتح الموقع لمدة، السيرفر "ينام" وأول طلب بعد النوم ياخذ 30-50 ثانية لين يصحى. طبيعي، مو خطأ.
- **قاعدة البيانات تنمسح مع كل نشر جديد** على الخطة المجانية (التخزين مؤقت). يعني لو عدّلت كود ورفعته مرة ثانية، كل الحسابات المسجلة تنمسح. لو حبيت تخلي البيانات دائمة، تحتاج تفعّل "Persistent Disk" (خدمة مدفوعة بسيطة) أو تنقل لاحقاً لقاعدة بيانات مستضافة مثل Render Postgres.
- تأكد إنك حدّثت `APP_BASE_URL` بالرابط الصحيح بعد أول نشر، وإلا رسائل التفعيل بترسل روابط خاطئة.
